## 2. প্রতিক্রিয়া ফ্রেমওয়ার্ক

### প্রস্তাবনা: Next.js 16 (অ্যাপ রাউটার)

|
**এই অ্যাপের জন্য Vite+React এর উপর Next.js কেন:**
- এসইও-এর জন্য এসএসআর/এসএসজি (সম্পদ পৃষ্ঠাগুলির জন্য গুরুত্বপূর্ণ যা অভিবাসী অভিভাবকদের গুগলে অনুসন্ধান করা উচিত)
- অন্তর্নির্মিত API রুটগুলি পৃথক ব্যাকএন্ড সার্ভারের প্রয়োজনীয়তা দূর করে
- রিঅ্যাক্ট সার্ভার কম্পোনেন্ট সহ অ্যাপ রাউটার = শূন্য ক্লায়েন্ট JS অনূদিত বিষয়বস্তুর জন্য
- `app/manifest.ts` এর মাধ্যমে নেটিভ PWA ম্যানিফেস্ট সমর্থন
- ভার্সেল স্থাপনা শূন্য-কনফিগ
- নেক্সট-ইনটেল RSC এর সাথে নেটিভভাবে কাজ করে (অনুবাদ রেন্ডার করা সার্ভার-সাইড = ক্লায়েন্টে 0 বাইট)

|
**যখন Vite ভাল হবে:**
- কোন SEO প্রয়োজন ছাড়া বিশুদ্ধ SPA
- দ্রুত ডেভ সার্ভার স্টার্টআপ (10-ঘন্টা হ্যাকাথনের জন্য প্রান্তিক সুবিধা)
- সহজ মানসিক মডেল (কোন সার্ভার/ক্লায়েন্ট উপাদান পার্থক্য নেই)

|
**সার্ভিস্টের সাথে PWA সেটআপ (পরবর্তী-pwa উত্তরাধিকারী):**
``বাশ
npm @servist/next @serwist/precaching @serwist/sw idb ইনস্টল করুন
```

|
মূল অফলাইন কৌশল:
- **ক্যাশফার্স্ট** স্ট্যাটিক সম্পদের জন্য (`/_next/static/`) -- বিষয়বস্তু-হ্যাশড, কখনই পরিবর্তন করবেন না
- API প্রতিক্রিয়া এবং গতিশীল সামগ্রীর জন্য **NetworkFirst**
- অনুবাদ ফাইল এবং সংস্থান পৃষ্ঠাগুলির জন্য **StaleWhileRevalidate**
- অফলাইন ডেটা স্টোরেজের জন্য IndexedDB (শিশু প্রোফাইল, সংরক্ষিত সম্পদ)

|
**গুরুত্বপূর্ণ:** Next.js 16 ডিফল্টরূপে Turbopack ব্যবহার করে, কিন্তু Serwist-এর জন্য Webpack প্রয়োজন। আপনাকে সেই অনুযায়ী বিল্ড কনফিগার করতে হবে।

|
**PWA ম্যানিফেস্ট** (বিল্ট-ইন Next.js সমর্থন):
`` টাইপস্ক্রিপ্ট
// app/manifest.ts
'পরবর্তী' থেকে { MetadataRoute } প্রকার আমদানি করুন

|
এক্সপোর্ট ডিফল্ট ফাংশন ম্যানিফেস্ট(): MetadataRoute.Manifest {
  ফিরে
    নাম: 'ASD ব্রিজ',
    সংক্ষিপ্ত_নাম: 'ASD ব্রিজ',
    বর্ণনা: 'অটিস্টিক শিশুদের সহ অভিবাসী পরিবারকে সহায়তা করা',
    start_url: '/',
    প্রদর্শন: 'স্বতন্ত্র',
    background_color: '#ffffff',
    theme_color: '#4F46E5',
    আইকন: [
      { src: '/icon-192.png', আকার: '192x192', টাইপ: 'image/png' },
      { src: '/icon-512.png', আকার: '512x512', টাইপ: 'image/png' },
    ],
  }
}
```

## 3. টাইপস্ক্রিপ্ট ব্যাকএন্ড

|
### সুপারিশ: Next.js API রুট + সুপাবেস ক্লায়েন্ট (প্রাথমিক) বা tRPC (যদি T3 ব্যবহার করেন)

|
**বিকল্প A: Next.js API রুট + সুপাবেস (হ্যাকাথনের জন্য সবচেয়ে সহজ)**
- কোন পৃথক ব্যাকএন্ড সার্ভার নেই
- সুপাবেস জেএস ক্লায়েন্ট ডিবি কোয়েরি, প্রমাণীকরণ, রিয়েলটাইম পরিচালনা করে
- এআই চ্যাট, অনুবাদ API কল এবং যেকোনো সার্ভার-সাইড লজিকের জন্য API রুট
- সেট আপ করার জন্য দ্রুততম

|
**বিকল্প B: T3 Stack এর মাধ্যমে tRPC (যদি দল এটি জানে তাহলে সেরা DX)**
- ফ্রন্টএন্ড এবং ব্যাকএন্ডের মধ্যে এন্ড-টু-এন্ড টাইপ নিরাপত্তা
- ফ্রন্টএন্ডে API কলগুলির জন্য স্বয়ংক্রিয়-সম্পূর্ণতা
- উৎপন্ন প্রকারের জন্য প্রিজমার সাথে কাজ করে
- `create-t3-app` সবকিছু ভারাক্রান্ত করে

|
``বাশ
# বিকল্প A: প্লেইন Next.js + Supabase
npx create-next-app@latest my-app --typescript --tailwind --eslint --app --src-dir

|
# বিকল্প B: T3 স্ট্যাক
npx create-t3-app@latest my-app
# নির্বাচন করুন: Next.js, TypeScript, Tailwind CSS, tRPC, Prisma
```

**কেন এক্সপ্রেস/ফাস্টফাই করবেন না:**
- স্থাপনার জটিলতা যোগ করে (হোস্টে আলাদা সার্ভার)
- এই ব্যবহারের ক্ষেত্রে Next.js API রুটের উপর কোন সুবিধা নেই
- সেটআপের অতিরিক্ত 30-60 মিনিট আপনার নেই

|
**কেন সার্ভারহীন নয় (ল্যাম্বডা/নেটলিফাই ফাংশন):**
- Vercel-এ Next.js API রুটগুলি ডিফল্টরূপে সার্ভারহীন
- আলাদা ফাংশন পরিকাঠামোর প্রয়োজন নেই

|

## 4. ডেটাবেস

|
### সুপারিশ: সুপাবেস (PostgreSQL)

|
**কেন সুপাবেস এই প্রকল্পের জন্য জিতেছে:**

|
| বৈশিষ্ট্য | সুপাবেস | ফায়ারবেস | প্ল্যানেটস্কেল |
|---------|----------|------------|-------------|
| বিনামূল্যের স্তর DB সঞ্চয়স্থান | 500 MB | 1 জিবি (স্পার্ক) | বন্ধ বিনামূল্যে স্তর |
| প্রমাণ অন্তর্ভুক্ত | হ্যাঁ (50K MAU বিনামূল্যে) | হ্যাঁ (50K MAU বিনামূল্যে) | না |
| রিয়েলটাইম | হ্যাঁ (পোস্টগ্রেস পরিবর্তন) | হ্যাঁ (শ্রেণির সেরা) | না |
| ভেক্টর অনুসন্ধান (pgvector) | হ্যাঁ, অন্তর্নির্মিত | না | না |
| এসকিউএল সমর্থন | সম্পূর্ণ PostgreSQL | NoSQL শুধুমাত্র | মাইএসকিউএল |
| RLS (রো লেভেল সিকিউরিটি) | হ্যাঁ, SQL-ভিত্তিক | ফায়ারবেস নিরাপত্তা নিয়ম | না |
| বেনামী প্রমাণীকরণ | হ্যাঁ, অন্তর্নির্মিত | হ্যাঁ | N/A |
| ওপেন সোর্স | হ্যাঁ | না | আংশিকভাবে |
| ডেটা বহনযোগ্যতা | সম্পূর্ণ (এটি পোস্টগ্রেস) | বিক্রেতা লক ইন | MySQL সামঞ্জস্যপূর্ণ |

**সুপাবেস ফ্রি টিয়ার (2026):**
- 2টি সক্রিয় প্রকল্প
- 500 MB ডাটাবেস স্টোরেজ
- 2 জিবি ডাটাবেস প্রস্থান
- 50,000 MAU (প্রমাণিকরণ)
- 1 জিবি ফাইল স্টোরেজ
- 500,000 এজ ফাংশন ইনভোকেশন
- কোন ক্রেডিট কার্ডের প্রয়োজন নেই, মেয়াদ শেষ হয় না

|
**এই অ্যাপের জন্য ফায়ারবেসের উপর সুপাবেস কেন:**
1. AI সম্পদ অনুসন্ধানের জন্য pgvector (কোন অতিরিক্ত পরিষেবার প্রয়োজন নেই)
2. জটিল প্রশ্নের জন্য সম্পূর্ণ SQL (শিশু মাইলফলক, দক্ষতা ট্র্যাকিং)
3. বেনামী ফোরাম অ্যাক্সেস গোপনীয়তা সংরক্ষণের জন্য RLS
4. ডেটা বহনযোগ্যতা (গুগল ইকোসিস্টেমে লক করা নেই)
5. রিলেশনাল ডেটার জন্য আরও ভাল (ব্যবহারকারী -> শিশু -> দক্ষতা -> মাইলফলক)

|
**প্রিজমা ইন্টিগ্রেশন:**
``প্রিজমা
ডেটাসোর্স ডিবি {
  প্রদানকারী = "postgresql"
  url = env("DATABASE_URL")
  directUrl = env("DIRECT_URL")
}
```

## 5. প্রমাণীকরণ

|
### সুপারিশ: বেনামী সাইন-ইন + ফোন ওটিপি + ইমেল/পাসওয়ার্ড সহ সুপাবেস প্রমাণীকরণ

|
**এই দর্শকদের জন্য সমালোচনামূলক ডিজাইনের নীতি:**
অ্যাপটি ব্যক্তিগতভাবে শনাক্তযোগ্য তথ্য প্রদান ছাড়াই ব্যবহারযোগ্য হতে হবে। অনেক অভিবাসী পিতামাতা (বিশেষত নথিভুক্ত) এমন একটি অ্যাপ ব্যবহার করবেন না যার জন্য সরকারি আইডি/SSN, আসল নাম যাচাইকরণ, ঠিকানা সংগ্রহ বা বাধ্যতামূলক ইমেল প্রয়োজন।

** প্রমাণীকরণ স্তর (প্রগতিশীল ট্রাস্ট):**

|
| স্তর | পদ্ধতি | এটা কি আনলক করে | PII আবশ্যক |
|------|---------|----------------|---------------|
| 1 | বেনামী সাইন ইন | সম্পদ ব্রাউজ করুন, এআই চ্যাট ব্যবহার করুন, ফোরাম দেখুন | কোনটিই না |
| 2 | ফোন ওটিপি | ফোরামে পোস্ট করুন, চাইল্ড প্রোফাইল সংরক্ষণ করুন | শুধুমাত্র ফোন নম্বর |
| 3 | ইমেল + পাসওয়ার্ড | পুনরুদ্ধারের সাথে সম্পূর্ণ অ্যাকাউন্ট | ইমেইল ঠিকানা |

|
**সুপাবেস বেনামী প্রমাণীকরণ:**
`` টাইপস্ক্রিপ্ট
// বেনামে সাইন ইন করুন -- কোন PII প্রয়োজন নেই৷
const { data, error } = await supabase.auth.signInAnonymously()

|
// পরে, ব্যবহারকারী একটি ফোন নম্বর লিঙ্ক করতে পারেন
const { data, error } = await supabase.auth.updateUser({
  ফোন: '+1234567890',
})
```

|
**ফোন ওটিপি সেটআপ:**
সুপাবেস Twilio, MessageBird, Textlocal এবং Vonage এর মাধ্যমে ফোন প্রমাণীকরণ সমর্থন করে। ব্যবহারকারীরা SMS এর মাধ্যমে একটি 6-সংখ্যার পিন পান, যা 60 সেকেন্ডের জন্য বৈধ।

|
**বেনামী ব্যবহারকারীদের জন্য নিরাপত্তা:**
- অপব্যবহার রোধ করতে ক্যাপচা (ক্লাউডফ্লেয়ার টার্নস্টাইল প্রস্তাবিত -- বিনামূল্যে) সক্ষম করুন৷
- আইপি-ভিত্তিক হারের সীমা: 30 অনুরোধ/ঘণ্টা (সুপাবেস ড্যাশবোর্ডে সামঞ্জস্যযোগ্য)
- RLS নীতিগুলি `is_anonymous` JWT দাবির মাধ্যমে বেনামী বনাম প্রমাণীকৃত ব্যবহারকারীদের পার্থক্য করতে পারে

|
**এই অ্যাপের জন্য কেরানি কেন নয়:**
- কোন অন্তর্নির্মিত বেনামী লগইন
- $0.02/MAU 10K পরে (সুপাবেস: 50K বিনামূল্যে)
- এই ব্যবহারের ক্ষেত্রে ওভারকিল UI
- বাহ্যিক নির্ভরতা যোগ করে

|
**কেন নয় প্রমাণ0:**
- হ্যাকাথনের জন্য জটিল সেটআপ
- কোন বেনামী auth
- বিনামূল্যের স্তর 7,500 MAU পর্যন্ত সীমাবদ্ধ

## 7. UI কম্পোনেন্ট লাইব্রেরি

|
### প্রস্তাবনা: shadcn/ui + Tailwind CSS

|
**কেন shadcn/ui:**
- উপাদানগুলি আপনার প্রকল্পে কপি-পেস্ট করা হয়েছে (সম্পূর্ণ মালিকানা, চিন্তা করার জন্য কোনও নির্ভরতা আপডেট নেই)
- Radix UI আদিম (WAI-ARIA অনুগত, কীবোর্ড নেভিগেশন, স্ক্রিন রিডার সমর্থন) উপর নির্মিত
- Tailwind CSS নেটিভ (RTL এর জন্য যৌক্তিক বৈশিষ্ট্য)
- 40+ উপাদান উপলব্ধ
- টেমপ্লেটের মাধ্যমে RTL সমর্থন
- গাঢ়/হালকা মোড অন্তর্নির্মিত
- শূন্য রানটাইম ওভারহেড

|
**ইনস্টলেশন:**
``বাশ
npx shadcn@latest init
npx shadcn@ সর্বশেষ যোগ বোতাম কার্ড ডায়ালগ ইনপুট ফর্ম শীট ট্যাব অবতার ব্যাজ অ্যাকর্ডিয়ান কমান্ড টোস্ট
```

|
**এই অ্যাপের মূল উপাদান:**
- `কার্ড` -- রিসোর্স কার্ড, স্কিল কার্ড, চাইল্ড প্রোফাইল কার্ড
- `ডায়ালগ` / `শীট` -- মডেল ইন্টারঅ্যাকশন, ভাষা পরিবর্তনকারী
- `ফর্ম` + `ইনপুট` -- চাইল্ড প্রোফাইল ফর্ম, ফোরাম পোস্ট তৈরি
- `ট্যাব` -- বিভাগগুলির মধ্যে নেভিগেশন
- `অবতার` -- ফোরাম ব্যবহারকারী প্রদর্শন (বেনামী বিকল্প সহ)
- `ব্যাজ` -- দক্ষতার মাত্রা, ভাষা ট্যাগ
- `অ্যাকর্ডিয়ন` -- প্রায়শই জিজ্ঞাসিত প্রশ্নাবলী, সম্পদের বিবরণ
- 'কমান্ড' -- সম্পদের জন্য অনুসন্ধান প্যালেট
- 'টোস্ট' -- বিজ্ঞপ্তি

|
**অভিগম্যতা বিল্ট-ইন:**
- সমস্ত রেডিক্স-ভিত্তিক উপাদানে স্বয়ংক্রিয়ভাবে ARIA ভূমিকা এবং গুণাবলী অন্তর্ভুক্ত
- কীবোর্ড নেভিগেশন বাক্সের বাইরে কাজ করে (ট্যাব, এন্টার, এস্কেপ, অ্যারো কী)
- মোডেলগুলিতে ফোকাস ব্যবস্থাপনা এবং ফোকাস ফাঁদ
- গতিশীল বিষয়বস্তুর জন্য স্ক্রীন রিডার ঘোষণা
- `aria-describedby` স্বয়ংক্রিয়ভাবে বৈধতা ত্রুটির সাথে সংযুক্ত
- ফর্ম ত্রুটির উপর `aria-অবৈধ` সেট

**চক্র UI কেন নয়:** ভারী রানটাইম (CSS-in-JS), প্রপ-ভিত্তিক স্টাইলিং টেইলউইন্ড ইউটিলিটি ক্লাসের তুলনায় কম নমনীয়, 2025-2026 সালে কম ইকোসিস্টেম গতিবেগ।

|
**মেটেরিয়াল UI কেন নয়:** খুব ভারী বান্ডিল, Google-এর ডিজাইন ভাষা একটি কমিউনিটি অ্যাপের জন্য ক্লিনিকাল মনে হতে পারে, গভীরভাবে কাস্টমাইজ করা কঠিন।

## 12. গোপনীয়তা এবং নিরাপত্তা

### গোপনীয়তা-বাই-ডিজাইন আর্কিটেকচার

|
**গাইডিং প্রিন্সিপল:** এই অ্যাপটি অরক্ষিত জনগোষ্ঠীর সেবা করে। গোপনীয়তা লঙ্ঘনের বাস্তব-বিশ্বের পরিণতি হতে পারে (অনথিভুক্ত পরিবারের জন্য নির্বাসনের ঝুঁকি)। নিরাপত্তা ঐচ্ছিক নয়।

|
**HIPAA বিবেচ্য বিষয়:** অ্যাপটি কোনো আচ্ছাদিত সত্তা নয় এবং সম্ভবত সম্পূর্ণ HIPAA সম্মতির প্রয়োজন নেই। যাইহোক, শিশুদের সম্পর্কে কোনো স্বাস্থ্য-সম্পর্কিত তথ্য সংরক্ষণ করলে, এটিকে সংবেদনশীল হিসাবে বিবেচনা করুন। সর্বোত্তম পদ্ধতি: আপনি সার্ভার-সাইডে যা সঞ্চয় করেন তা ছোট করুন।

|
**COPPA বিবেচনা (13 বছরের কম বয়সী শিশু):** যদি পিতামাতারা একমাত্র ব্যবহারকারী হন, তবে COPPA কম সীমাবদ্ধ কিন্তু এখনও শিশুদের ডেটা সংরক্ষণের জন্য প্রাসঙ্গিক। 2025 COPPA আপডেট কঠোর ডেটা মিনিমাইজেশন প্রয়োজনীয়তা প্রবর্তন করে। **প্রস্তাবিত:** অ্যাপটি শুধুমাত্র পিতামাতার জন্য ডিজাইন করুন, শিশুদের সরাসরি ব্যবহারের জন্য নয়।

|
**ডেটা আর্কিটেকচার -- কোথায় কি সঞ্চয় করতে হবে:**

| ডেটা টাইপ | স্টোরেজ অবস্থান | এনক্রিপশন |
|------------|-----------------|------------|
| সন্তানের নাম | ক্লায়েন্ট-সাইড (স্থানীয় স্টোরেজ/ইনডেক্সডডিবি) | ঐচ্ছিক ক্লায়েন্ট-সাইড AES-256 |
| শিশুর রোগ নির্ণয় | ক্লায়েন্ট-সাইড | AES-256 ক্লায়েন্ট-সাইড এনক্রিপশন |
| শিশুর দক্ষতা/মাইলস্টোন | সুপাবেস (বিশ্রামে এনক্রিপ্ট করা) | সুপাবেস ডিফল্ট |
| ফোরাম পোস্ট | সুপাবেস | বিশ্রামে (সুপাবেস ডিফল্ট) |
| এআই চ্যাট ইতিহাস | শুধুমাত্র ক্লায়েন্ট-সাইড (সেশন স্টোরেজ) | ক্ষণস্থায়ী, স্থায়ী নয় |
